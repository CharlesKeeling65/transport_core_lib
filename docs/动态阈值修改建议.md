# 动态阈值修改建议：方案 A 与方案 B 完整分析

## 一、当前问题分析

### 1.1 现有实现的三个根本缺陷

现有的 `calculate_dynamic_threshold`（第 36-49 行）本质上是一个分段常函数：

```python
density < 0.1 → threshold = 32
0.1 ≤ density < 0.3 → threshold = 40
0.3 ≤ density < 0.5 → threshold = 48
density ≥ 0.5 → threshold = 60
```

存在以下三个根本问题：

1. **硬编码的分段边界与乘数缺乏理论依据**
   - 为什么是 0.1/0.3/0.5？为什么乘数是 0.8/1.0/1.2/1.5？
   - 这些数字完全是经验猜测，没有基于算法的实际复杂度分析。

2. **阶梯函数的不连续性**
   - $\rho=0.299$ 和 $\rho=0.301$ 的数据在分布上几乎一致，但阈值却从 40 跳变到 48。
   - 这种不连续性会导致算法选择的突然变化，影响实验结果的可复现性。

3. **未利用 $n_{\mathrm{jobs}}$ 参数**
   - 函数接收了 `n_jobs` 参数但从未使用。
   - 并行度显著影响 KD-Tree 方法的相对性能，8 核和 32 核机器的切换点应该不同。

### 1.2 时间估算函数的问题

现有的 `estimate_traditional_method_time` 和 `estimate_adaptive_method_time` 两个函数也存在建模不准确的问题：

- `estimate_traditional_method_time` 使用 `n_offsets * n_supply * n_demand / 1e6`，这与实际代码不符（实际代码用数组操作，复杂度更接近 $O(d \cdot XY)$）。
- 尽管如此，我们可以从算法的实际复杂度出发，建立更准确的成本模型。

## 二、共同的理论基础：两种算法的成本模型

### 2.1 传统方法的真实时间复杂度

传统方法（`_traditional_method`）对 `direct_list(d)` 生成的每个偏移量执行三个全栅格数组操作：

```python
# 每个偏移量 (dr, dc) 执行:
moved_p    = move_array(supply_raster, (dr, dc), distance)   # pad + roll + crop → O(XY)
edges      = edges_line_func(moved_p, demand_raster)          # element-wise AND → O(XY)  
edge_rows, edge_cols = np.nonzero(edges)                      # 扫描非零元素 → O(XY)
```

`direct_list(d)` 生成的是圆环上的整数格点，其数量 ≈ 圆环面积：

$$
\begin{aligned}
N_{\mathrm{off}}(d) &= \pi(d + \varepsilon)^2 - \pi(d - 1 + \varepsilon)^2,\quad \varepsilon = \frac{\sqrt{2}}{2} \\
&= \pi\bigl[(d^2 + 2\varepsilon d + \varepsilon^2) - (d^2 - 2d + 2\varepsilon d + 1 - 2\varepsilon + \varepsilon^2)\bigr] \\
&= \pi(2d - 1 + 2\varepsilon) \\
&\approx 2\pi d \qquad (d \gg 1)
\end{aligned}
$$

每个偏移量产生约 7 次 $O(XY)$ 的 numpy 操作（pad→roll→astype→!=0×2→&→nonzero），因此：

$$
\begin{aligned}
T_{\mathrm{traditional}}(d) &= c_{\mathrm{arr}} \cdot N_{\mathrm{off}}(d) \cdot XY \cdot 7 \\
&\approx c_{\mathrm{arr}} \cdot 2\pi d \cdot XY \cdot 7 \\
&= \alpha \cdot d \cdot XY,\qquad \alpha = 14\pi\, c_{\mathrm{arr}}
\end{aligned}
$$

**关键特征**：$T_{\mathrm{traditional}}$ 与数据密度无关，只取决于栅格总像素和搜索半径。

### 2.2 KD-Tree 方法的真实时间复杂度

`_kd_tree_method` 先分块，每块中调用 `process_grid_chunk_v2`：

```python
# 每块中:
tree = cKDTree(demand_coords)                    # O(D_c log D_c)
idx_outer = tree.query_ball_point(supply_coords, max_radius)  # O(S_c log D_c)  
idx_inner = tree.query_ball_point(supply_coords, min_radius)
# set difference + edge building: O(S_c × avg_result_size)
```

$$
\begin{aligned}
T_{\mathrm{kdtree}}(S, D) &= c_{\mathrm{build}} \cdot D\log_2 D + c_{\mathrm{query}} \cdot (S\log_2 D + D\log_2 D) + c_{\mathrm{edge}} \cdot N_{\mathrm{edges}} \\
&\approx \beta \cdot \frac{(S+D)\,\log_2(S+D)}{n_{\mathrm{jobs}}} \qquad \text{（主导项合并简化）}
\end{aligned}
$$

其中 $\beta$ 是包含构建、查询、集合操作的有效机器常数。

**关键特征**：$T_{\mathrm{kdtree}}$ 与点数 $N = S+D$ 成 $O(N\log N)$ 关系，与栅格总大小 $XY$ 无关。

## 三、方案 A：直接成本比较器（Cost-Based Optimizer）

### 3.1 数学原理

不求解 $d^\star$，而是直接估算两种方法在当前输入下的预期耗时，取最小值。即：对每条查询，做一次**成本查询**而非一次阈值比较。

$$
\text{Choice} = \operatorname*{arg\,min}\bigl\{ T_{\mathrm{traditional}}(d, XY),\, T_{\mathrm{kdtree}}(S, D, n_{\mathrm{jobs}}) \bigr\}
$$

其中：

$$
\begin{aligned}
T_{\mathrm{traditional}}(d, XY) &= c_{\mathrm{arr}} \cdot 2\pi d \cdot XY \cdot 7 \approx C_t \cdot d \cdot XY \\
T_{\mathrm{kdtree}}(S, D) &= c_{\mathrm{kdt}} \cdot \frac{(S+D)\,\log_2(S+D)}{n_{\mathrm{jobs}}}
\end{aligned}
$$

$C_t$ 和 $c_{\mathrm{kdt}}$ 是机器校准常数。

### 3.2 代码实现

```python
import math


def estimate_traditional_time(
    distance: int,
    n_supply: int,
    n_demand: int,
    total_pixels: int,
    C_t: float = 3e-9,       # 校准常数：d × XY 项的系数
) -> float:
    """
    估算传统方法执行时间（秒）。
    
    理论基础: T ≈ 2πd × XY × 7 × c_arr = C_t × d × XY
    其中 C_t = 14π × c_arr, c_arr 是单元素操作时间。
    """
    if distance <= 0 or total_pixels <= 0:
        return 0.0
    return C_t * distance * total_pixels


def estimate_kdtree_time(
    n_supply: int,
    n_demand: int,
    n_jobs: int = 8,
    c_kdt: float = 5e-8,     # 校准常数：(S+D)log(S+D) 项的系数
) -> float:
    """
    估算 KD-Tree 方法执行时间（秒）。
    
    理论基础: T ≈ c_kdt × (S+D) × log₂(S+D) / n_jobs
    """
    total = n_supply + n_demand
    if total == 0 or n_jobs <= 0:
        return 0.0
    log_n = math.log2(total) if total > 1 else 1.0
    return c_kdt * total * log_n / n_jobs


def select_faster_method(
    n_supply: int,
    n_demand: int,
    distance: int,
    n_jobs: int,
    total_pixels: int,
    C_t: float = 3e-9,
    c_kdt: float = 5e-8,
) -> str:
    """选择成本更低的方法（无阈值概念）"""
    t_trad = estimate_traditional_time(distance, n_supply, n_demand, total_pixels, C_t)
    t_kdt  = estimate_kdtree_time(n_supply, n_demand, n_jobs, c_kdt)
    return 'traditional' if t_trad <= t_kdt else 'kd_tree'
```

与上段 **docstring** 中文字描述等价的 LaTeX 记法如下：

$$
T_{\mathrm{traditional}} \approx 2\pi d \cdot XY \cdot 7 \cdot c_{\mathrm{arr}} = C_t \cdot d \cdot XY,\qquad C_t = 14\pi\, c_{\mathrm{arr}}
$$

$$
T_{\mathrm{kdtree}} \approx c_{\mathrm{kdt}} \cdot \frac{(S+D)\,\log_2(S+D)}{n_{\mathrm{jobs}}}
$$

### 3.3 集成到现有框架

```python
# 在 Hybrid_TKD 分支中替换 threshold 比较逻辑
if algorithm == AlgorithmType.HYBRID_TKD:
    method = select_faster_method(n_supply, n_demand, distance, n_jobs, supply_raster.size)
    if method == 'traditional':

    else:
        elapsed, edges = self._kd_tree_method(supply_raster, demand_raster, distance, Y, n_jobs)
```

### 3.4 校准方案

两个常数 $C_t$ 和 $c_{\mathrm{kdt}}$ 有明确的物理含义，校准更透明：

$$
C_t = \frac{T_{\mathrm{Trad,\,measured}}}{d \cdot XY} \qquad \text{（只需一次传统法测量）}
$$

$$
c_{\mathrm{kdt}} = \frac{T_{\mathrm{kdtree,\,measured}} \cdot n_{\mathrm{jobs}}}{(S+D)\,\log_2(S+D)} \qquad \text{（一次 KD-Tree 测量）}
$$

也可加入运行时自适应：

```python
# 每次运行后，用实际观测值更新常数（指数移动平均）
C_t = 0.9 * C_t + 0.1 * (t_trad_actual / (d × XY))
c_kdt = 0.9 * c_kdt + 0.1 * (t_kdt_actual * n_jobs / (total * log_n))
```

对应的指数移动平均（EMA）更新式为：

$$
\begin{aligned}
C_t &\leftarrow 0.9\, C_t + 0.1 \cdot \frac{t_{\mathrm{trad}}^{\mathrm{actual}}}{d \cdot XY} \\
c_{\mathrm{kdt}} &\leftarrow 0.9\, c_{\mathrm{kdt}} + 0.1 \cdot \frac{t_{\mathrm{kdt}}^{\mathrm{actual}} \cdot n_{\mathrm{jobs}}}{\mathrm{total} \cdot \text{log\_n}}
\end{aligned}
$$

（其中 $\mathrm{total}$、`log_n` 与上一段 Python 中的变量一致；`log_n` 在实现里为 $\log_2(\mathrm{total})$（当 $\mathrm{total}>1$）否则为 $1$。）

## 四、方案 B：解析阈值法（Analytical Threshold）

### 4.1 数学推导

令两种方法耗时相等，求解 $d^\star$：

$$
\alpha \cdot d^\star \cdot XY = \beta \cdot \frac{(S+D)\,\log_2(S+D)}{n_{\mathrm{jobs}}}
$$

整理：

$$
\begin{aligned}
d^\star &= \frac{\beta}{\alpha} \cdot \frac{(S+D)\,\log_2(S+D)}{XY \cdot n_{\mathrm{jobs}}} \\
&= K \cdot \frac{\rho\,\log_2(S+D)}{n_{\mathrm{jobs}}}
\end{aligned}
$$

其中：

- $\rho = (S+D)/XY$——点在栅格中的占空比（无量纲）（文中亦记作 density）
- $K = \beta/\alpha$——机器校准常数（唯一需要确定的参数，具有距离量纲）

### 4.2 公式的直观解读

| 条件 | $d^\star$ 变化 | 决策偏向 | 直觉 |
|------|----------------|---------|------|
| $\rho \uparrow$ | $d^\star \uparrow$ | 偏向传统法 | 传统法成本由 XY 主导（不变）；KD-Tree 需处理更多点 |
| $S{+}D \uparrow$（固定 $XY$） | $d^\star \uparrow$ | 偏向传统法 | 同上 |
| $n_{\mathrm{jobs}} \uparrow$ | $d^\star \downarrow$ | 偏向 KD-Tree | 分块并行天然适合 KD-Tree |
| $XY \uparrow$（固定 $S{+}D$） | $d^\star \downarrow$ | 偏向 KD-Tree | 传统法每偏移量扫描整个大栅格，成本线性增长 |
| $d < d^\star$ | — | 用传统法 | 小距离下传统法的 $O(d)$ 优势明显 |
| $d \ge d^\star$ | — | 用 KD-Tree | 大距离下 KD-Tree 的 $O(\log N)$ 查询优势压倒 $O(d)$ |

### 4.3 代码实现（已实际应用）

```python
import math
import numpy as np


def calculate_dynamic_threshold(
    n_supply,
    n_demand,
    n_jobs=8,
    total_pixels=44102097,
    K=150.0,
    d_min=10,
    d_max=200,
):
    """
    基于成本模型交叉点的解析动态阈值。
    
    数学推导：
        T_traditional ≈ c1 × d × XY
        T_kdtree ≈ c2 × (S+D) × log(S+D) / n_jobs
        令 T_traditional = T_kdtree，解出：
        d* = (c2/c1) × (S+D) × log(S+D) / (XY × n_jobs)
           = K × density × log(S+D) / n_jobs
    
    参数：
        K: 机器校准常数，默认值 150 在典型硬件上适用。
           K 越小代表 KD-Tree 相对越快（如 MKL 加速的机器），
           K 越大代表传统法相对越快（如小缓存/慢内存的机器）。
        d_min: 下限保护，防止极端稀疏场景下阈值过低。
        d_max: 上限保护，防止极端密集场景下阈值过高。
    """
    total_points = n_supply + n_demand
    if total_points == 0 or total_pixels <= 0 or n_jobs <= 0:
        return d_min
    
    density = total_points / total_pixels
    log_n = math.log2(total_points) if total_points > 1 else 1.0
    
    d_star = K * density * log_n / max(n_jobs, 1)
    
    return int(np.clip(d_star, d_min, d_max))
```

与函数实现一致的闭合形式（对应 docstring 中的推导）：

$$
d^\star = K \cdot \frac{\rho \cdot \log_2(S+D)}{n_{\mathrm{jobs}}},\qquad \rho = \frac{S+D}{XY}
$$

### 4.4 使用方式（与现有代码兼容）

```python
# Hybrid_TKD 中的决策逻辑不变，仅替换函数体
threshold = calculate_dynamic_threshold(
    n_supply, n_demand, n_jobs, supply_raster.size, K=150
)
if distance <= threshold:
    elapsed, edges = self._traditional_method(...)
else:
    elapsed, edges = self._kd_tree_method(...)
```

### 4.5 K 的校准方案

校准 K 需要一次"交叉点定位"实验：

1. 在目标数据集上，固定 $\rho$（density）和 $n_{\mathrm{jobs}}$
2. 对多个距离 $d$ 分别运行传统法和 KD-Tree 法
3. 找到 $T_{\mathrm{traditional}} \approx T_{\mathrm{kdtree}}$ 时的 $d_{\mathrm{observed}}$
4. $K = \dfrac{d_{\mathrm{observed}} \cdot n_{\mathrm{jobs}}}{\rho \cdot \log_2(S+D)}$

也可以直接用已有的时间估算函数自动校准：

```python
def auto_calibrate_K(supply_raster, demand_raster, n_jobs, d_search=40):
    """自动扫描确定最优 K"""
    n_supply = np.count_nonzero(supply_raster)
    n_demand = np.count_nonzero(demand_raster)
    total_pixels = supply_raster.size
    density = (n_supply + n_demand) / total_pixels
    log_n = math.log2(n_supply + n_demand) if n_supply + n_demand > 1 else 1
    
    # 在 d_search 附近对比两种方法的实际耗时
    t_trad, _ = _traditional_method(supply_raster, demand_raster, d_search)
    t_kdt, _ = _kd_tree_method(supply_raster, demand_raster, d_search)
    
    # 如果 t_trad ≈ t_kdt，交叉点在 d_search 附近
    # 否则根据比值推算交叉点
    d_estimated = d_search * math.sqrt(t_kdt / t_trad)
    K = d_estimated * n_jobs / (density * log_n)
    return K
```

其中比例启发式来自 $d^\star \propto T_{\mathrm{kdtree}} / T_{\mathrm{traditional}}$（在交叉点附近近似）。

## 五、方案 A 与方案 B 的完整对比

| 维度 | 方案 A：成本比较器 | 方案 B：解析阈值 |
|------|-------------------|----------------|
| **数学模型** | $\operatorname*{arg\,min}\{T_{\mathrm{trad}}, T_{\mathrm{kdtree}}\}$ | $d^\star = K \cdot \rho \cdot \log_2 N \,/\, n_{\mathrm{jobs}}$ |
| **输出** | 算法名称 | 阈值距离 $d^\star$ |
| **运行时计算量** | 两次简单算术运算 | 一次简单算术运算 |
| **校准常数** | 2 个（$C_t$, $c_{\mathrm{kdt}}$） | 1 个（$K$） |
| **常数物理含义** | $C_t$：传统法每 $d \cdot XY$ 秒数 | $K = \beta/\alpha$（两方法机器常数比） |
| | $c_{\mathrm{kdt}}$：KD-Tree 每 $N\log N$ 秒数 | |
| **校准方式** | 各一次单一实验即可确定 | 需要定位交叉点实验 |
| **对分布非均匀的鲁棒性** | 中等（成本模型本身不建模分布） | 中等（同为成本模型的产物） |
| **扩展性** | **高**：加新算法只需加新 `estimate_*` 函数 | **低**：需重新推导阈值公式 |
| **与现有代码兼容** | 需修改 `run_single_experiment` 的 hybrid 分支 | **直接替换原函数即可** |
| **异常值敏感性** | 常数通过 EMA 在线更新，更鲁棒 | $K$ 校准不当会系统性偏移 |
| **可解释性** | 透明的成本估算，适合论文实验部分 | 阈值 $d^\star$ 直观，适合论文方法部分 |
| **实现难度** | 低（约 30 行新代码） | 极低（约 15 行，替换原函数） |

## 六、数值敏感性对比

为了直观展示两个方案的行为差异，下面计算在不同的 $\rho$（density）和 $d$ 条件下两种方案的决策结果。

假设固定 $XY = 44\mathrm{M}$，$n_{\mathrm{jobs}} = 8$，校准后 $K = 150$，$C_t = 3\times 10^{-9}$，$c_{\mathrm{kdt}} = 5\times 10^{-8}$。

|--------|---------|-----------------|------------------------|---------|

| 0.01 | 440K | 18.7 | 3.5 → clamp=10 | 大部分用 KD-Tree |

| 0.50 | 22M | 24.4 | 228 → clamp=200 | 几乎全用传统法 |

| $\rho$ $\downarrow$ / $d$ $\rightarrow$ | 10 | 20 | 40 | 70 | 100 |

**关键结论**：两个方案的决策边界是**一致的**——因为方案 B 的阈值 $d^\star$ 正是解 $T_{\mathrm{trad}} = T_{\mathrm{kdtree}}$ 得到的解析解。区别在于方案 B 通过一次预计算将决策函数简化为 “$d \le d^\star$？”，而方案 A 在每次调用时重新计算成本。

## 八、结论

两个方案的核心创新在于用基于成本模型的解析解替代了经验分段函数。方案 B 的优势是改动最小、直接可替换；方案 A 的优势是更可扩展、校准更透明。两者在内核上是等价的——方案 B 的 $d^\star$ 正是方案 A 中 $T_{\mathrm{trad}} = T_{\mathrm{kdtree}}$ 的解。

**当前已实施**：方案 B（解析阈值公式）已替换原有的分段函数，提交到 transport_core_lib（commit: 3ff0f1a）并同步到主仓库。
